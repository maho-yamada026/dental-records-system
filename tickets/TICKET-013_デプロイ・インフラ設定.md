# TICKET-013: デプロイ・インフラ設定

## 概要
本番・ステージング環境を構築し、継続的デプロイを設定する。

## 推奨構成
- **ホスティング**: Vercel（Next.js）または Render
- **DB**: Supabase（PostgreSQL）または Railway
- **ファイルストレージ**: 不要（CSVはメモリ上で処理）

> 小規模（数百名の患者）かつ社内利用のため、マネージドサービスで十分。

## 環境
| 環境 | 用途 |
|---|---|
| development | ローカル開発（docker-compose） |
| staging | 動作確認・クライアントレビュー |
| production | 本番運用 |

## タスク
- [ ] Vercel プロジェクト作成・GitHubリポジトリ連携
- [ ] Supabase プロジェクト作成・本番DB設定
- [ ] 環境変数（DATABASE_URL, JWT_SECRET等）をVercelに設定
- [ ] staging ブランチへのPush → ステージング自動デプロイ設定
- [ ] mainブランチへのマージ → 本番自動デプロイ設定
- [ ] Prisma マイグレーションの本番適用手順ドキュメント化
- [ ] 本番DBバックアップ設定（Supabase自動バックアップ確認）
- [ ] 独自ドメイン設定（必要であれば）

## セキュリティ確認
- [ ] HTTPS強制確認
- [ ] JWT_SECRET が十分にランダムな値か確認（32文字以上）
- [ ] DBのパブリックアクセス制限（Vercelのみ許可）
- [ ] 本番ログに個人情報が出力されないこと確認

## 完了条件
- staging URLでアプリが動作する
- mainマージ後、5分以内に本番に反映される
- DBは暗号化・定期バックアップが有効
