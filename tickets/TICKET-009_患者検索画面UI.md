# TICKET-009: 患者検索画面UI

## 概要
ドクターが患者を素早く検索・選択できる画面を実装する。タブレット操作を前提とした設計にする。

## 画面仕様

### 患者検索画面 `/patients`

**検索フォーム（常時表示）**
- 患者IDテキスト入力（大きめのフォント・タップしやすいサイズ）
- 氏名・カナ テキスト入力
- 医院選択ドロップダウン
- 治療部位選択（チェックボックス または タグ選択）
- 検索ボタン / リアルタイム検索（500ms デバウンス）

**検索結果リスト**
- 患者カード形式で表示（患者ID・氏名・カナ・医院・最終診療日）
- ページネーション（または無限スクロール）
- 各カードをタップ → 患者詳細画面へ遷移

## UI要件（タブレット最適化）
- タップターゲット最小 44x44px
- フォント: 最小 16px（老眼対応）
- キーボード表示時にもフォームが見えるようスクロール制御
- 横向き（landscape）・縦向き（portrait）両方に対応
